## 詳細

### V017-001 カメラモーション読み込み時に位置やタイミングがズレる
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: カメラ VMD 読み込み, 再生, フレームシーク, 最終フレーム停止, タイムライン表示
- 現象:
  - カメラモーションを読み込んだとき、想定したカメラ位置やタイミングからズレるという報告が複数ある
  - 特に最終フレーム付近でズレや評価不整合が出ている可能性がある
- メモ:
  - まずは同じ VMD を本家 MMD と MMD_modoki で比較し、先頭フレーム、中間キー、最終キー、最終フレーム後停止時のカメラ状態を確認する
  - `duration` の扱い、`currentFrame` の丸め、`lastFrame` の決定、最終キーと再生停止フレームの off-by-one を疑う
  - カメラ VMD 読み込み直後、タイムラインシーク時、再生終了時で `MmdCamera -> ArcRotateCamera` 同期タイミングが異なる可能性も確認する
  - v0.1.3 の `V013-009` はカメラキーフレーム登録経路の修正であり、今回は外部カメラモーション読み込み / 評価の別件として扱う

### V017-002 WebM 動画出力が始まらない
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: WebM 出力, MediaRecorder, canvas capture, 出力 UI, ファイル保存
- 現象:
  - WebM 動画出力を開始しようとしても、出力処理が始まらないという報告がある
  - エラー表示の有無、保存ダイアログの有無、録画中 UI へ遷移するかは未確認
- メモ:
  - `MediaRecorder.isTypeSupported` と codec 選択の結果を確認する
  - canvas capture stream の取得可否、WebGPU / WebGL2 経路差、UI 非表示モードとの干渉を切り分ける
  - 開始前 validation、保存先パス、出力中状態フラグ、例外ログの出方を確認する

### V017-003 物理演算が MMD 本家と違って少しかたい
- 種類: bug
- 優先度: P2
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: 物理演算, Ammo 設定, 剛体, joint, 再生安定化, MMD 互換性
- 現象:
  - 同じモデル / モーションでも、MMD 本家より物理演算の挙動が少しかたく見えるという報告がある
  - 髪、スカート、揺れ物など、どの部位で目立つかは未切り分け
- メモ:
  - MMD 本家との比較動画または同一フレームのスクリーンショットで差分を確認する
  - 物理モード 0/1/2、`disableOffsetForConstraintFrame`、`disableBidirectionalTransformation` の設定差を確認する
  - timestep、substep、初期姿勢安定化、再生開始時のリセット手順が挙動のかたさに影響していないか見る
  - `docs/mmd-basic-task-checklist.md` の「物理モード 0/1/2 の比較検証」と関連する

### V017-004 Mac 版で FPS が安定しない
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: macOS ビルド, 描画, 再生, FPS, GPU, Electron
- 現象:
  - Mac 版で FPS が安定しないという報告がある
  - 一定して遅いのか、周期的に落ちるのか、特定機能で落ちるのかは未確認
- メモ:
  - v0.1.6 の `V016-001`「macOS 版でパフォーマンスが遅い」からの継続案件として扱う
  - WebGPU / WebGL2 の実行経路、GPU 利用状況、影設定、DoF / SSAO / Bloom などのポストエフェクト負荷を切り分ける
  - 背景動画読み込み後、`.x` アクセサリ表示時、影プリセット変更時など v0.1.7 追加領域との関連も確認する

### V017-005 カメラが原点から離れると近接モデル材質や床が欠けやすい
- 種類: bug
- 優先度: P1
- 発生日: 2026-04-13
- 送信元: ユーザー報告
- 状態: investigating
- 影響範囲: カメラ表示, viewport, clipping, frustum culling, 床表示, モデル材質描画
- 現象:
  - カメラが原点から離れた状態で、カメラに近接したモデルの材質が消えやすい
  - 同じ条件で床も欠けやすい
- メモ:
  - v0.1.6 の `V016-004`「画面端でポリゴンが欠ける」と関連する可能性はあるが、今回は「原点から離れたカメラ位置」「近接モデル材質」「床欠け」を主条件として別件管理する
  - `minZ / maxZ`、near clip、far clip、projection precision、frustum culling、bounding info 更新を切り分ける
  - 床 mesh の bounds、camera target / radius、MMD カメラ同期時の position / target / distance の意味ずれも確認する
  - 再現時はカメラ座標、target、distance、FOV、WebGPU / WebGL2 経路、床表示 ON/OFF、対象モデルの材質名を記録する
